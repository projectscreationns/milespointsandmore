# Miles, Points and More

A fast, free, self-updating replacement for the old Wix site. Static HTML generated by a tiny
Python script (no frameworks, no paid tools), hosted for $0, and refreshed weekly by an automated
agent that finds new AU/US points & miles deals and publishes them **in the author's voice**.

## How it works

```
content/        the site's data — you (or the weekly agent) only ever edit this
  site.json     title, tagline, nav, socials, subscribe embed
  posts/*.json  one file per article: title, date, tags, summary, body_html
  pages/*.html  About / Contact (+ matching .json for the title)
templates/*.html  page skeletons ({{ tokens }} get filled in)
assets/         style.css + favicon (copied as-is into the build)
build.py        reads content/ + templates/ -> writes the finished site into public/
STYLE.md        the author's voice guide — the weekly agent MUST follow this
automation/weekly-post.md  the exact job the weekly agent runs
public/          BUILD OUTPUT (git-ignored; the host regenerates it)
```

Nothing else. To add a post, drop a JSON file in `content/posts/`. To change the look, edit
`assets/style.css`. To change wording/nav, edit `content/site.json`.

## Build & preview locally

Requires Python 3.9+ (standard library only — nothing to install).

```bash
python build.py            # generates ./public
python build.py --serve    # generates, then serves at http://localhost:8000
```

> Note: on the original Windows/OneDrive authoring folder, native tools can't write into the
> folder, so build locally by copying the repo elsewhere or just let the host build it. On any
> normal checkout (Linux/Mac, GitHub, Cloudflare) `python build.py` works directly.

## Hosting (free)

**Cloudflare Pages (recommended):**
1. Push this repo to GitHub.
2. Cloudflare dashboard → Pages → Connect to Git → pick the repo.
3. Build command: `python build.py` · Output directory: `public`.
4. Deploy. You get `something.pages.dev` free. Every `git push` redeploys.
5. Add `milespointsandmore.com` under the project's Custom Domains and update DNS.

**GitHub Pages (fallback):** the included `.github/workflows/deploy.yml` builds and deploys on
every push to `main`. Enable Pages → Source: GitHub Actions.

## The weekly automation

A scheduled Claude Code agent runs `automation/weekly-post.md` once a week. It reads `STYLE.md`,
researches the latest AU/US deals, writes one new post in the author's voice, self-reviews it,
rebuilds, and pushes — which auto-deploys. Posts go live with no manual review, so every post
carries the standard "not financial advice" disclaimer (added automatically in the footer).

## Adding your own post by hand

Copy any file in `content/posts/`, change the fields, rebuild. `date` is `YYYY-MM-DD`. Use single
quotes for HTML attributes inside `body_html`.
